# 대규모 언어 모델 코드 생성 비교 연구

## 연구 개요

본 연구는 대규모 언어 모델(LLM)들의 코드 생성 능력을 체계적으로 비교 분석하는 것을 목표로 합니다. 동일한 요구사항을 제시하여 각 모델이 생성한 코드의 품질, 구조, 완성도를 다각도로 평가합니다.

## 비교 대상 모델

- **GPT 5.2**
- **Gemini 3 Pro** 
- **DeepSeek V3**
- **Claude Sonnet 4.5**

## 연구 방법론

### 1단계: 추상적 요구사항 기반 코드 생성
각 모델에 추상적인 요구사항을 제시하여 코드를 생성하고, 모델별 해석 및 구현 방식의 차이점을 분석합니다.

### 2단계: 구체적 요구사항 기반 코드 생성  
1단계와 동일한 도메인에 대해 더욱 구체적이고 상세한 요구사항을 제시하여 코드를 생성하고, 세부 명세에 대한 각 모델의 대응 능력을 평가합니다.

### 3단계: 비교 분석
1단계와 2단계에서 생성된 코드를 종합적으로 비교하여 각 모델의 강점과 약점을 도출합니다.

## 연구 도메인

**무인 판매기 장치 등록 시스템 (Device Registration System)**

- 기술 스택: TypeScript, Node.js, NestJS
- 주요 기능: 장치 등록, 관리자 승인/거부, 상태 관리, 이벤트 처리
- 장치 상태: UNREGISTERED → PENDING → REGISTERED → DECOMMISSIONED

## 요구사항 문서

### 1차 요구사항 (추상적)
- 파일: `prompt/1차 요구사항.txt`
- 특징: 기본적인 기능 요구사항과 도메인 개념 정의
- 구현 자유도: 높음

### 2차 요구사항 (구체적)
- 파일: `prompt/2차 요구사항.txt`  
- 특징: 상세한 API 명세, 인증/인가, 응답 형식, 이벤트 처리 등 구체적 명시
- 구현 자유도: 낮음

### 비교 평가 기준
- 파일: `prompt/비교 프롬프트.txt`
- 8개 평가 항목 (입력 검증, 에러 처리, 보안, 로직 디테일 등)
- 가중치 적용 점수 시스템 (1~5점)

## 평가 항목

1. **입력 검증** (가중치: 1.5) - 필수값, 데이터 형식, 검증 위치, 오류 메시지
2. **에러 처리** (가중치: 1.4) - 예외 처리, HTTP 상태 코드, 로깅, 응답 구조
3. **보안 요소** (가중치: 1.4) - 인증/인가, 접근 제한, 시큐어 코딩
4. **로직 디테일** (가중치: 1.3) - 요구사항 구현, 가독성, 문서화
5. **로직 분리 구조** (가중치: 1.2) - 계층 분리, 의존성 주입, 모듈화
6. **API URI 및 메소드** (가중치: 1.1) - RESTful 설계, HTTP 메소드
7. **응답 구조** (가중치: 1.0) - 일관성, 명명 규칙
8. **확장성** (가중치: 0.8) - 미래 대응성, 설정 분리

## 연구 결과

### 1차 요구사항 결과
- 각 모델별 생성 코드 분석
- 추상적 요구사항에 대한 해석 차이점
- 기본 아키텍처 설계 접근법 비교

### 2차 요구사항 결과  
- 구체적 명세에 대한 준수도 평가
- 세부 기능 구현 완성도 비교
- 코드 품질 및 구조적 완성도 분석

### 종합 비교 분석
- 1차 vs 2차 요구사항 대응 능력 변화
- 모델별 강점 및 약점 도출
- 실무 적용 관점에서의 유용성 평가

## 연구 의의

이 연구를 통해 다음과 같은 인사이트를 도출할 수 있습니다:

- 각 LLM의 코드 생성 특성 및 강점 영역 파악
- 요구사항 명세 수준에 따른 모델별 성능 변화 분석  
- 실무 프로젝트에서의 LLM 활용 가이드라인 제시
- 향후 LLM 기반 개발 도구 개선 방향 제안
